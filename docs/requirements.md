# URLSender 要件定義

## 目的

- 複数端末間でURLを共有し、再検索の手間をなくす
- 共有したURLを早めに読む習慣をつける

## 想定ユーザー

- 自分のみ（将来のサービス化は保留。悪用リスクの懸念があるため）

## 対応端末

- PC・スマホ・タブレット横断。できるだけ多くの組み合わせに対応する

## 開発フェーズ

- **Phase 1**: Webアプリ + ブラウザ拡張機能（送信用）、認証機能
- **Phase 2**: スマホアプリ（プッシュ通知対応）

## 機能要件

- **URL送信**: ブラウザ拡張機能・アプリ（Web）のどちらからも送信可能にする
- **URL一覧表示**: 送られたURLを一覧で確認できる
- **ログイン認証**: しっかりした認証を自作する（学習目的も兼ねる）
- **履歴**: 送信履歴を保持する
- **メモ**: 各URLに自由記述のメモを残せる
- **既読/未読**: ユーザーの手動切り替えのみ（自動既読化はしない）
- **データ保持**: デフォルトで14日経過後に自動削除する。ただし「保存/ピン留め」したものは自動削除の対象外とする
- **通知**: Phase 1では実装しない。Phase 2でスマホアプリのプッシュ通知に対応する

## データ項目

- **URL**: 送信されたURL本文
- **送信元/送信先端末情報**: どの端末から送られたか（任意、履歴表示の参考情報）
- **送信日時**
- **メモ**: 自由記述のテキスト
- **既読/未読ステータス**
- **保存/ピン留めフラグ**: 自動削除の対象外にするかどうか
- **ユーザー情報**: ログイン用のアカウント情報（自分専用の1ユーザーのみ）

## 非機能要件

- **セキュリティ（認証）**
  - パスワードは平文で保存せず、ハッシュ化（bcryptなど）して保存する
  - セッション/トークンには有効期限を設け、期限切れ後は再ログインを要求する
  - ログイン試行回数の制限など、簡易的な総当たり対策を入れる
- **セキュリティ（データ）**
  - 通信は必ずHTTPSを使用する
  - APIキーやパスワードなどの秘密情報はコードにハードコードしない（`.env`管理、`.gitignore`対象）
- **可用性/性能**
  - 個人利用規模のため高負荷対策は不要。ただし複数端末からの同時アクセスに問題なく応答できること
- **保守性**
  - 将来的なサービス化（複数ユーザー対応）を見据え、ユーザーIDに紐づく設計にしておく（ただし複数ユーザー機能自体はスコープ外）

## スコープ外（今回は対応しない）

- 複数ユーザーへのサービス提供（招待制・アカウント発行など）
- ブラウザへのプッシュ通知
- URLの自動既読化
