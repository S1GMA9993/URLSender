# URLSender 要件定義

## 目的

- 複数端末間でURLを共有し、再検索の手間をなくす
- 共有したURLを早めに読む習慣をつける

## 想定ユーザー

- 自分のみ（将来のサービス化は保留。悪用リスクの懸念があるため）

## 対応端末

- PC・スマホ・タブレット横断。できるだけ多くの組み合わせに対応する

## 開発フェーズ

- **Phase 1**: Webアプリ + ブラウザ拡張機能（送信用）、認証機能
- **Phase 2**: スマホアプリ（プッシュ通知対応）

## 機能要件

- **URL送信**: ブラウザ拡張機能・アプリ（Web）のどちらからも送信可能にする
- **URL一覧表示**: 送られたURLを一覧で確認できる
- **ログイン認証**: しっかりした認証を自作する（学習目的も兼ねる）
- **履歴**: 送信履歴を保持する
- **メモ**: 各URLに自由記述のメモを残せる
- **既読/未読**: ユーザーの手動切り替えのみ（自動既読化はしない）
- **データ保持**: デフォルトで14日経過後に自動削除する。ただし「保存/ピン留め」したものは自動削除の対象外とする
- **通知**: Phase 1では実装しない。Phase 2でスマホアプリのプッシュ通知に対応する
